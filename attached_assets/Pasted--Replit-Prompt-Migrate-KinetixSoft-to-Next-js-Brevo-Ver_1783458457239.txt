# Replit Prompt — Migrate KinetixSoft to Next.js + Brevo + Vercel

Paste this into Replit. This is a full migration from the current Vite/React + Express setup to a Next.js App Router project, with Brevo contact form integration and Vercel-ready configuration.

---

## What we're doing and why
The current setup is a Vite React frontend + separate Express API server. This means:
- Crawlers (Google, GPTBot, ClaudeBot etc.) see a blank page — SEO/AEO is broken
- Replit hosting is not production-grade (sleep mode, no custom domain properly)
- Contact form has no email backend

We're migrating to **Next.js 14 App Router** which gives us:
- Server-side rendering → every page's HTML is visible to crawlers on first load
- API routes built in → no separate Express server needed
- Vercel-optimized → deploys perfectly with zero config
- Brevo API route → contact form emails go to inbox

## Step 6 — Contact form API route with Brevo (`src/app/api/contact/route.ts`)

```ts
import { NextRequest, NextResponse } from 'next/server'

export async function POST(req: NextRequest) {
  const body = await req.json()
  const { name, email, phone, country, message } = body

  if (!name || !email || !message) {
    return NextResponse.json({ error: 'Missing required fields' }, { status: 400 })
  }

  try {
    const response = await fetch('https://api.brevo.com/v3/smtp/email', {
      method: 'POST',
      headers: {
        'accept': 'application/json',
        'content-type': 'application/json',
        'api-key': process.env.BREVO_API_KEY!,
      },
      body: JSON.stringify({
        sender: { name: 'KinetixSoft Website', email: 'noreply@kinetixsoft.com' },
        to: [{ email: process.env.CONTACT_EMAIL!, name: 'KinetixSoft Team' }],
        replyTo: { email, name },
        subject: `New inquiry from ${name}`,
        htmlContent: `
          <h2>New Contact Form Submission</h2>
          <table style="border-collapse:collapse; width:100%">
            <tr><td style="padding:8px;border:1px solid #eee;"><strong>Name</strong></td><td style="padding:8px;border:1px solid #eee;">${name}</td></tr>
            <tr><td style="padding:8px;border:1px solid #eee;"><strong>Email</strong></td><td style="padding:8px;border:1px solid #eee;">${email}</td></tr>
            <tr><td style="padding:8px;border:1px solid #eee;"><strong>Phone</strong></td><td style="padding:8px;border:1px solid #eee;">${phone || '—'}</td></tr>
            <tr><td style="padding:8px;border:1px solid #eee;"><strong>Country</strong></td><td style="padding:8px;border:1px solid #eee;">${country || '—'}</td></tr>
            <tr><td style="padding:8px;border:1px solid #eee;"><strong>Message</strong></td><td style="padding:8px;border:1px solid #eee;">${message}</td></tr>
          </table>
        `,
      }),
    })

    if (!response.ok) {
      const err = await response.json()
      console.error('Brevo error:', err)
      return NextResponse.json({ error: 'Failed to send email' }, { status: 500 })
    }

    return NextResponse.json({ success: true })
  } catch (err) {
    console.error(err)
    return NextResponse.json({ error: 'Server error' }, { status: 500 })
  }
}
```

## Step 7 — ContactForm component (`src/components/ContactForm.tsx`)

```tsx
'use client'
import { useState } from 'react'

export default function ContactForm() {
  const [status, setStatus] = useState<'idle' | 'loading' | 'success' | 'error'>('idle')

  async function handleSubmit(e: React.FormEvent<HTMLFormElement>) {
    e.preventDefault()
    setStatus('loading')
    const form = e.currentTarget
    const data = {
      name: (form.elements.namedItem('name') as HTMLInputElement).value,
      email: (form.elements.namedItem('email') as HTMLInputElement).value,
      phone: (form.elements.namedItem('phone') as HTMLInputElement).value,
      country: (form.elements.namedItem('country') as HTMLInputElement).value,
      message: (form.elements.namedItem('message') as HTMLTextAreaElement).value,
    }

    const res = await fetch('/api/contact', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data),
    })

    setStatus(res.ok ? 'success' : 'error')
  }

  return (
    <form onSubmit={handleSubmit}>
      <input name="name" type="text" placeholder="Full Name" required />
      <input name="email" type="email" placeholder="Email Address" required />
      <input name="phone" type="tel" placeholder="Phone Number" />
      <input name="country" type="text" placeholder="Country" />
      <textarea name="message" placeholder="Tell us about your business" rows={4} required />
      <button type="submit" disabled={status === 'loading'}>
        {status === 'loading' ? 'Sending...' : 'Submit'}
      </button>
      {status === 'success' && <p className="success-msg">Message sent — we'll be in touch shortly.</p>}
      {status === 'error' && <p className="error-msg">Something went wrong. Please email us directly.</p>}
    </form>
  )
}
```

## Step 8 — JSON-LD schema component (`src/components/JsonLd.tsx`)

```tsx
export default function JsonLd({ data }: { data: object }) {

    <script
      type="application/ld+json"
```

## Step 9 — robots.txt and sitemap

Create `src/app/robots.ts`:
```ts
import type { MetadataRoute } from 'next'
export default function robots(): MetadataRoute.Robots {
  return {
    rules: [
      { userAgent: '*', allow: '/' },
      { userAgent: 'GPTBot', allow: '/' },
      { userAgent: 'ClaudeBot', allow: '/' },
```

## Step 10 — Environment variables

Create `.env.local` in the project root:
```
BREVO_API_KEY=your_brevo_api_key_here
CONTACT_EMAIL=your@email.com
```

**How to get Brevo API key:**
1. Sign up at brevo.com (free)
2. Go to Settings → API Keys → Generate a new key
3. Paste it in `.env.local`

**Never commit `.env.local` to Git** — add it to `.gitignore` (Next.js does this automatically).

For each of the 6 pages, copy the existing JSX/HTML content from the current Vite build into the corresponding Next.js `page.tsx` file. Key rules:
- Add `export const metadata = pageMetadata.pageName` at the top of each file

## Step 12 — Deploy to Vercel

1. Push the project to GitHub:
```bash
git init
git add .
git commit -m "Initial Next.js migration"
git remote add origin https://github.com/YOUR_USERNAME/kinetixsoft-web.git
git push -u origin main
```

2. Go to vercel.com → New Project → Import from GitHub → select `kinetixsoft-web`

3. In Vercel project settings → Environment Variables, add:
   - `BREVO_API_KEY` = your Brevo API key
   - `CONTACT_EMAIL` = your@email.com

4. Deploy — Vercel auto-detects Next.js, zero config needed.

5. In Vercel → Domains → add `kinetixsoft.com` → update your domain DNS to point to Vercel's nameservers.
