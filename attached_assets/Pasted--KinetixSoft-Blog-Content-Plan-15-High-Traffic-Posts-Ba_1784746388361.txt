## On-Page SEO Rules for Every Blog Post

1. **H1** = exact blog title (once only)
2. **H2s** = use secondary keywords naturally ("FlutterFlow development cost", "no-code app builder 2026")
3. **First 100 words** = include the primary keyword in the first sentence or two
4. **Meta description** = use exactly as written above (under 160 characters)
5. **URL slug** = exactly as written above (no dates in URL)
6. **Word count** = 1,200–2,000 words per post minimum
7. **Internal links** = link to the relevant platform page (e.g., FlutterFlow blog post → /flutterflow page)
8. **CTA at end** = every post ends with a clear next step (contact, book a call, or link to a service page)
9. **Author** = "KinetixSoft Team" or a real named team member

## Blog Page Setup for Replit

When implementing the blog in the Next.js project, structure it as:

- `/blog` — listing page with all posts, category filter tabs
- `/blog/[slug]` — individual post page (already exists in codebase)
- Categories available as filter tabs: All, FlutterFlow, Bubble, Lovable, Replit, Retool, Podio, Comparisons, How To's, Guides
- Each post stored as a content object or MDX file in `src/content/blog/`
- Sitemap updated to include all blog post URLs

Each blog post needs:
```tsx
export const metadata: Metadata = {
  title: "Post Title | KinetixSoft",
  description: "Meta description from above",
  alternates: { canonical: `https://kinetixsoft.com/blog/${slug}` },
  openGraph: {
    title: "Post Title",
    description: "Meta description",
    url: `https://kinetixsoft.com/blog/${slug}`,
    type: "article",
    publishedTime: "2026-07-20",
    authors: ["KinetixSoft Team"],
  },
};
```
