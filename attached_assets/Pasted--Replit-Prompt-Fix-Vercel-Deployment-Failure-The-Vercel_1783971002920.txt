# Replit Prompt — Fix Vercel Deployment Failure

The Vercel deployment is failing at `npm install` with:
`npm error Exit handler never called! Error: Command "npm install" exited with 1`

This is caused by a `preinstall` script in `package.json` that blocks npm and forces pnpm.
Vercel uses npm — so the build dies immediately. Fix everything below in order.

---

## FIX 1 — Remove the preinstall script from package.json (CRITICAL)

Open `package.json` and find this script:
```json
"preinstall": "sh -c 'rm -f package-lock.json yarn.lock; case \"$npm_config_user_agent\" in pnpm/*) ;; *) echo \"Use pnpm instead\" >&2; exit 1 ;; esac'"
```

**Delete it completely.** The final `scripts` section must look exactly like this:

```json
{
  "name": "kinetixsoft-web",
  "version": "0.1.0",
  "private": true,
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "next lint"
  },
  "dependencies": {
    "next": "14.2.5",
    "react": "^18.3.1",
    "react-dom": "^18.3.1"
  },
  "devDependencies": {
    "@types/node": "^20",
    "@types/react": "^18",
    "@types/react-dom": "^18",
    "typescript": "^5",
    "tailwindcss": "^3.4.1",
    "autoprefixer": "^10.4.19",
    "postcss": "^8",
    "eslint": "^8",
    "eslint-config-next": "14.2.5"
  }
}
```

No `preinstall`, no `pnpm` references, no workspace scripts. Just Next.js.

---

## FIX 2 — Remove pnpm-related files (if still present)

```bash
rm -f pnpm-lock.yaml
rm -f pnpm-workspace.yaml
rm -f .npmrc
```

If `.npmrc` exists and contains `engine-strict=true` or `only-allow=pnpm`, delete that file entirely. These force pnpm and break Vercel.

---

## FIX 3 — Generate a clean package-lock.json for npm

Vercel needs a `package-lock.json` to do fast installs. Generate it:

```bash
rm -f package-lock.json
rm -rf node_modules
npm install
```

This creates a fresh `package-lock.json` using npm. Commit this file to git — Vercel uses it to speed up builds.

---

## FIX 4 — Add vercel.json to the project root

Create a `vercel.json` file in the project root:

```json
{
  "framework": "nextjs",
  "buildCommand": "npm run build",
  "installCommand": "npm install",
  "outputDirectory": ".next"
}
```

This tells Vercel exactly how to build — no guessing.

---

## FIX 6 — Remove unused Firebase configs from the project

The project has Firebase config variables (FIREBASE_APP_ID, FIREBASE_AUTH_DOMAIN, etc.) but Firebase is not being used. Remove any Firebase import from the codebase:

```bash
# Check if firebase is imported anywhere
grep -r "firebase" src --include="*.ts" --include="*.tsx" --include="*.js" -l
```

If any files are found, remove the firebase imports from them. Firebase is not needed — the site is static + Brevo only.

Also remove firebase-admin from package.json if it appears in dependencies:
```bash

```

## FIX 9 — Add CONTACT_EMAIL to Vercel (do this manually)

Go to Vercel → kinetix-soft project → Settings → Environment Variables → Add:

| Key | Value | Environment |
|---|---|---|
| `BREVO_API_KEY` | (already set ✅) | Production + Preview |
