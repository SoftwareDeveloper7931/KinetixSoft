## What we're implementing and why

| Type | Full Name | What it does |
|---|---|---|
| **SEO** | Search Engine Optimization | Google, Bing — traditional crawl-and-rank |
| **AEO** | Answer Engine Optimization | Google Featured Snippets, PAA boxes — direct question answers |
| **GEO** | Generative Engine Optimization | Perplexity, ChatGPT Search — AI that cites sources |
| **AIO** | AI Indexing Optimization | GPTBot, ClaudeBot, Gemini — AI training & retrieval crawlers |

Each requires a different approach. All four are implemented below.

## PART 2 — AEO (Answer Engine Optimization)

AEO targets Google Featured Snippets, "People Also Ask" boxes, and voice search. The goal is to have your FAQ answers and section intros pulled directly as snippet answers.

### 2A. FAQPage schema — already in `FaqSection.tsx` ✅

The `FaqSection` component already injects `FAQPage` JSON-LD. Verify every platform page passes its full `faqs` array to `<FaqSection faqs={faqs} />`.

### 2B. Service schema — add to each platform page

Add inside each platform `page.tsx` alongside the page component:

```tsx
// In flutterflow/page.tsx
const serviceSchema = {
  "@context": "https://schema.org",
  "@type": "Service",
  "name": "FlutterFlow App Development",
  "provider": { "@type": "Organization", "name": "KinetixSoft" },
  "areaServed": "Worldwide",
  "description": "Custom FlutterFlow app development for startups and growing teams, including UI/UX design, AI features, third-party integrations, and ongoing support.",
  "offers": { "@type": "Offer", "availability": "https://schema.org/InStock" }
};
// Repeat with correct name + description for: Podio, Lovable, Replit, Retool
```

Pass it via the `serviceSchema` prop into `<PlatformPageLayout serviceSchema={serviceSchema} ... />` — the layout already renders it via `<JsonLd data={p.serviceSchema} />` ✅

### 2C. Review schema — add to each platform page case study

Add alongside the service schema:

```tsx
const reviewSchema = {
  "@context": "https://schema.org",
  "@type": "Review",
  "itemReviewed": { "@type": "Organization", "name": "KinetixSoft" },
  "reviewRating": { "@type": "Rating", "ratingValue": "5", "bestRating": "5" },
  "author": { "@type": "Person", "name": "Amir Khan" }, // update per page
  "reviewBody": "We came in with a complicated forecasting model and KinetixSoft shipped something our users genuinely understand and enjoy using." // update per page
};
```
Then add `<JsonLd data={reviewSchema} />` inside the page component.

### 2D. BreadcrumbList schema — add to each platform page

### 2E. Direct-answer structure — verify in all section bodies

For AEO, the first sentence after a heading must directly answer the implied question. Audit these specific spots:

- "Why We Build on [Platform]" section → first sentence must answer "Why use [platform]?" directly, before any supporting detail.
- FAQ answers → must be 1–3 sentences, self-contained. Do not start with "Great question" or any filler.
- Process section → each step title should be a verb phrase ("We map your goals" not just "Discovery").

### 2F. Structured list markup — verify `<ul>/<li>` not `<div>`

All bullet lists (features, process bullets, "why platform" bullets) must use real `<ul><li>` HTML, not styled `<div>` elements. AI answer engines and Google parse list structure to build snippet answers.

## PART 3 — GEO (Generative Engine Optimization)

GEO targets Perplexity, ChatGPT Search, Google AI Overviews, and other AI tools that synthesize answers from the web and cite sources. The goal is to be cited when someone asks "which agency should I use for FlutterFlow" or "best Retool developer."

### 3A. Consistent entity description — use word-for-word everywhere

This exact description must appear identically in: the Organization schema, the homepage meta description (trimmed), and the homepage "about" copy. Repetition across multiple pages signals the same entity to AI crawlers:

> **"KinetixSoft is an app development studio that builds production-ready mobile, web, and internal tools using FlutterFlow, Lovable, Retool, Podio, and Replit. Founded in 2025, KinetixSoft works with startups and growing teams who need a working product fast without sacrificing scalability or security."**

Verify this text appears verbatim in `src/app/layout.tsx` (Organization schema `description` field) and in the homepage body copy.

### 3B. Factual, specific bullet points — audit "Why We Build on [Platform]" sections

These bullet lists are the most likely content to be cited in an AI comparison answer (e.g. Perplexity answering "best FlutterFlow agency"). They must be:
- **Specific**, not generic. "Faster build cycles than traditional development" ✅. "We are passionate about quality" ❌.
- **Factual claims**, not marketing adjectives.
- **Distinct per platform** — Podio bullets should not read like FlutterFlow bullets.

### 3C. No duplicate paragraphs across pages

Run this check: the homepage body, the FlutterFlow page body, and the Podio page body must not share any sentences word-for-word. Duplicate content dilutes entity authority in generative AI citation. Each platform page should describe only what is unique about that platform engagement.

## PART 4 — AIO (AI Indexing Optimization)

AIO targets the crawlers AI companies use to build their training datasets and retrieval indexes — GPTBot (OpenAI), ClaudeBot (Anthropic), GoogleBot-Extended (Gemini), PerplexityBot, CCBot (Common Crawl). The goal is to ensure these crawlers can read your pages and that your content is structured to be retrieved accurately.

### 4A. robots.ts — `src/app/robots.ts`

Create or replace this file:

```ts
import type { MetadataRoute } from "next";

export default function robots(): MetadataRoute.Robots {
  return {
    rules: [
      { userAgent: "*", allow: "/" },
      { userAgent: "GPTBot", allow: "/" },
      { userAgent: "ChatGPT-User", allow: "/" },
      { userAgent: "ClaudeBot", allow: "/" },
      { userAgent: "anthropic-ai", allow: "/" },
      { userAgent: "PerplexityBot", allow: "/" },
      { userAgent: "Google-Extended", allow: "/" },
      { userAgent: "CCBot", allow: "/" },
      { userAgent: "Amazonbot", allow: "/" },
      { userAgent: "FacebookBot", allow: "/" },
      { userAgent: "Applebot", allow: "/" },
    ],
    sitemap: "https://kinetixsoft.com/sitemap.xml",
  };
}
```

### 4D. Verify server-side rendering

AI crawlers (like Google-Extended, ClaudeBot) do not execute JavaScript. Because this is Next.js with App Router and all platform pages are server components (no `"use client"` at the page level), every page's HTML is fully rendered on the server before it reaches the crawler. ✅

**One thing to check:** `ContactForm` and `FaqSection` use `"use client"`. That's fine — their surrounding page content (headings, body copy, schema) is still server-rendered. But make sure FAQ questions are also present in the server-rendered HTML, not only injected after JS runs. Since `FaqSection` is client-side, add a no-JS fallback: render all FAQ questions as a `<noscript>` block with the same content.

Add to `FaqSection.tsx`:
```tsx
<noscript>
  <div>
    {faqs.map((f, i) => (
      <div key={i}>
        <h3>{f.q}</h3>
        <p>{f.a}</p>
      </div>
    ))}
  </div>
</noscript>
```
